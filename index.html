Dataset Overview¶

The dataset below a preprocessed version of the The Ultimate 1Million Movies Dataset (TMDB + IMdb) by Alan Vourc'h. The processing script can be found at oridim/ds220-pandas-project/process_datasets.py. All it does is:

  • Project specific needed fields for the analysis found in this Jupyter notebook.
  • Strip out rows with missing column data.
  • Cleans up column data to remove any potential needed error handling from the Jupyter notebook.
  • Normalize column keys.
  • Limit the rows to only English (the language, not the British) movies that have been already released.
  • Limit the rows to only movies that had a theatrical release (i.e. no made-for-TV movies).

The already processed dataset can be found at oridim/ds220-pandas-project/datasets/oridim.movies.cleaned.csv

def format_runtime(minutes):
    """Formats minute runtime lengths into X hrs Y mins."""

    minutes = int(minutes)
    hours = minutes // 60
    minutes = minutes % 60

    return f"{hours} hrs {minutes} mins"


def format_output(df):
    """
    A formatting help to tell Pandas to format columns representing
    money and runtime durations into user-friendly strings.
    """

    return df.style.format(
        {
            "budget": "$ {:,.2f}",
            "box_office": "$ {:,.2f}",
            "rating": "{:.2f}",
            "runtime": format_runtime,
        }
    )

Framing of the Analysis¶

None of the questions being asked in this notebook are really of much importance. This analysis does not provide world-changing healthcare insights, for instance. However, these of just some questions I have always wondered about film statistics. And this seemed like the perfect time to figure some of these out.

Live Action vs. Animation¶

Animated movies are often thought to some that they generate more buzz due to factors such as:

  • More artistically driven given the crew behind are creating a whole world from scratch.
  • More financial reach due to the parents of the youth culture-driven model of filmmaking.
  • etc.

Is this true, however?

Distribution¶

Q: What is the distribution of live-action films vs. animation films?

Animation is generally a more work intensive process. Not only do artists have to create their own world from scratch, they also have to physically create it from scratch. Features like furry textures on animals or the reflection of light off shiny surfaces require the artists to put in detailed work. Compared to just recording those things happening naturally with a camera.

Therefore, for the expected return vs. box office revenue, I expect there to be significantly less animated movies than there are live-action.

has_animation = df["genres"].apply(lambda genres: "animation" in genres)


def render_top_averages(n):
    """
        Renders a bar chart showing the top-N box office average revenues
        between each category of filmmaking.
    """

    # Create a new grouped dataframe which calculates the average of
    # animated and live-action movies, respectively.
    # 
    # The new dataframe only takes the largest n number of titles from
    # each category. 
    box_office_averages = df.groupby(has_animation)["box_office"].apply(
        lambda series: series.nlargest(n).mean()
    )

    box_office_averages.index = box_office_averages.index.map(
        {
            True: "Animated Movies",
            False: "Live Action Movies",
        }
    )

    # Render a bar chart to show the raw averages of each category.
    plot = box_office_averages.plot.bar(
        title=f"Live Action vs. Animation Box Office Averages (Top {n})",
        xlabel="",
        ylabel="Average Box Office ($)",
        rot=0,
    )

    # To create a "zoomed in" bar chart, we are setting the limits of
    # the y-axis to the maximum and minimum averages. Along with some
    # minor padding for a sense of scale.
    min_average = box_office_averages.min()
    max_average = box_office_averages.max()

    y_range = max_average - min_average
    y_padding = y_range * 0.10

    plot.set_ylim(bottom=min_average - y_padding, top=max_average + y_padding)

    # Before showing the bar chart, we need to set up money formatting
    # for the y-axis and in-chart values. 
    formatter = mtick.StrMethodFormatter("${x:,.2f}")
    plot.yaxis.set_major_formatter(formatter)

    for container in plot.containers:
        plot.bar_label(container, fmt="$ {:,.2f}")

    plt.tight_layout()
    plt.show()
